Solution #1 递归

方法的思路就是每次走到当前的位置 i， 然后根据 nums[i] 的值，来决定下一跳可能跳到的位置。遍历每一种跳法到达下一个位置，然后重复的之前的方法，去尝试所以可能的一下跳的位置。直到我们发现下一跳的位置已经超过end， 我们就返回0。

在返回的时候，我们根据所有可能性的跳法，返回那格跳的次数最少的答案+1返回，因为要加上本身自己的这一跳。

我们定义一个递归函数helper， 这个函数返回的值是从当前start的位置开始，跳到最后end的最少次数。这个递归的退出条件是当前位置start已经超过end，这个时候返回的最少跳数是0.还有一个退出条件是当前start位置上的值是0，也就是说我们无法继续再往下跳了，所以返回的最少跳数是 float(‘inf’), 也就是无限。

如果当前的start位置没有超过end，而且还可以继续往下跳，那么我们就继续调用递归函数去尝试所有的下一跳的可能性，然后找到那个跳数最少的那一个，最后加上自己本身的一跳返回。

我们发现这个方法中，递归函数有许多重复计算的部分，比如说，当我们计算start=5的时候，我们到达5这个位置有许多种方式。可以是从位置1跳4步到达5，或者是从位置2跳3步到达5，每次我们到达5这个位置的之后，我们为了要计算接下来最少的步数，我们还要继续计算，但是这个计算在这两种情况下面是被重复计算了两次。为了避免这种重复的计算，我们用了一个数组dp，来保存每次计算结果。这样我们下次到达同样一个位置的时候，我们可以直接从保存结果dp里面拿答案。

time complexity: O(n)O(n)O(n)﻿ space complexity: O(n)O(n)O(n)﻿ n代表的是nums的长度

Code

class Solution(object):
    def jump(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        # helper function to return the
        # the min steps that reaches to the end
        def helper(start, dp):
            if start >= len(nums)-1:
                return 0
            if dp[start] != -1:
                return dp[start]
            # if current num is 0, means cannot reach to the end
            if nums[start] == 0:
                dp[start] = float('inf')
            else:
                # try with different steps and return the min steps
                dp[start] = min([helper(start+i, dp) for i in xrange(1, nums[start]+1)]) + 1
            return dp[start]
        # dp used for skip the duplicated calculation
        dp = [-1]*len(nums)
        return helper(0, dp)

Solution #2 贪心法

贪心法的思路是我们总是根据我们已经能够到达的位置cur去找到下一跳我们能找到的最远的位置next。当我们发现我们当前的这一跳cur没有办法到达下一个位置的时候i，我们就增加一跳 jumps+1，然后以后我们之前已经计算好的下一跳最远的距离next来替换当前一跳最远的距离cur。当我们发现我们当前这一条的最远距离cur已经涵盖了所有的位置len(nums)之后，就返回之前保存的跳的次数+1， 因为我们每次拿当前跳最远距离cur去和当前位置i做判断，但是并没有真正的跳。

这里的一个corner case就是nums的长度小于或等于1，因为我们已经在index=0上的位置了，随意我虽然做了判断，但是我们也不需要去跳，所以就不需要jump+1

time complexity: O(n)O(n)O(n)﻿ space complexity: O(1)O(1)O(1)﻿ n代表的是nums的长度

Code

class Solution(object):
    def jump(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        # nums is None or len(nums) < 2, we don't have to jump
        # since we are already in index 0
        if not nums or len(nums) < 2:
            return 0
        # cur: the furthest position for current jump
        # next: the furthest position for next jump
        next = cur = nums[0]
        # count number of jumps
        jumps = 0
        # iterate each position. use the cur to compare
        # with current position to ensure we can reach here.
        # if current jump cannot reach here, we need make one more jump
        # to get further reachable position from next jump
        for i in xrange(0, len(nums)):
            if cur < i:
                jumps += 1
                cur = next
            # we need to find furthest position within
            # all of our reachable positions(gready strategy)
            next = max(next, i+nums[i])
        # finally we have to add 1 more jump in answer
        # because we only compare cur with current position
        # to know if we can reachable, but not really jump.
        return jumps+1

Solution #3 followup

这道题目说我们一定能够到达最后的终点，也就是说我们不需要考虑我们有可能不能到达终点的情况。但是如果我们需要考虑不能到达终点的情况呢？如果题目要求我们在不能到达终点的情况下，我们返回-1

其实这种情况下的解题思路也是一样的，只不过我们需要额外添加一个判断的情况。

首先我们考虑在什么情况下我们会发现我们永远不可能到达终点？一定是我们发现当前一跳不能够到达下一个位置的时候。在solution2的情况下，我们就会用下一跳的最远距离next来替换当前一跳的最远距离cur。这里我们默认下一跳的最远距离next一定可以到达下一个位置i。所以我们需要加的判断也就是在更新当前这一跳最远距离cur的时候，判断下一跳最远距离next能不能到达下一个位置i。如果不可以的话，就说明我们永远没有办法到达终点。也就返回-1

time complexity: O(n)O(n)O(n)﻿ space complexity: O(1)O(1)O(1)﻿

Code

class Solution(object):
    def jump(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        # nums is None or len(nums) < 2, we don't have to jump
        # since we are already in index 0
        if not nums or len(nums) < 2:
            return 0
        # cur: the furthest position for current jump
        # next: the furthest position for next jump
        next = cur = nums[0]
        # count number of jumps
        jumps = 0
        # iterate each position. use the cur to compare
        # with current position to ensure we can reach here.
        # if current jump cannot reach here, we need make one more jump
        # to get further reachable position from next jump
        for i in xrange(0, len(nums)):
            if cur < i:
                jumps += 1
                # if furthest position in next jump
                # cannot reach to current position
                # it means we never be able to reach
                # to the end position
                if next < i:
                    return -1
                cur = next
            # we need to find furthest position within
            # all of our reachable positions(gready strategy)
            next = max(next, i+nums[i])
        # finally we have to add 1 more jump in answer
        # because we only compare cur with current position
        # to know if we can reachable, but not really jump.
        return jumps+1
