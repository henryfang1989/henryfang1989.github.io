Solution #1

Use DFS with memorization search

  1. build up the graph to allow us get the next stop quickly
  1. start with src and keep iterating all next stops.
  1. If current stop is same as dst, return the cost
  1. If K <= 0, which mean we run out of stops, just return float('inf')
  1. we save the search results with src and k into dp to prevent duplicated calculation.
  1. for dp, we have to save k in dp as well to indicate how many steps left with current starting city src

time complexity: O(n^2+k*n) space complexity: O(n^2+n*k)

Code

class Solution(object):
    def findCheapestPrice(self, n, flights, src, dst, K):
        """
        :type n: int
        :type flights: List[List[int]]
        :type src: int
        :type dst: int
        :type K: int
        :rtype: int
        """
        # build the graph
        prices = [[-1 for i in xrange(n)] for j in xrange(n)]
        for c1, c2, p in flights:
            prices[c1][c2] = p
        
        # find lowest prices for src to dst with up to k steps
        def helper(prices, src, dst, K, dp):
            if dp[K-1][src] != -1:
                return dp[K-1][src]
            if src == dst:
                return 0
            if K <= 0:
                return float('inf')
            res = float('inf')
            for nxt in xrange(n):
                if prices[src][nxt] != -1:
                    res = min(res, helper(prices, nxt, dst, K-1, dp) + prices[src][nxt])
            dp[K-1][src] = res
            return res
        dp = [[-1 for i in xrange(n)] for j in xrange(K+1)]
        res = helper(prices, src, dst, K+1, dp)
        return res if res != float('inf') else -1

Solution #2

This approach is using BFS to expand the trips from starting city src

  1. build up the graph to allow us to find the next stop
  1. Create a array dp to save lowest price by arriving the current city from src
  1. start with src and expand to the next stops
  1. If a lower price with arriving the same city, update the lower price in dp, also add that city to queue to keep expanding
  1. Until we reach to K steps, we should have the lowest price for dst
  1. Every time, we expand one more step, we have create a new copy of dp, because we shouldn't update the lowest price in previous dp because the stop in the same step may use the updated price, which messes up the steps info.

time complexity: O() space complexity: O()

Code

class Solution(object):
    def findCheapestPrice(self, n, flights, src, dst, K):
        """
        :type n: int
        :type flights: List[List[int]]
        :type src: int
        :type dst: int
        :type K: int
        :rtype: int
        """
        # build up the graph
        hashmap = collections.defaultdict(list)
        for a, b, c in flights:
            hashmap[a].append([b, c])
            
        dp = [float('inf') for i in xrange(n)]
        dp[src] = 0
        q = collections.deque([src])
        K += 1
        while q and K:
            # create a copy of dp for updating
            # cannot reuse the old dp array, which may mess up with steps
            cur_dp = dp[:]
            sz = len(q)
            K -= 1
            for _ in xrange(sz):
                cur = q.popleft()
                for nxt, price in hashmap[cur]:
                    # find lower price, update price in new copy of dp
                    if dp[cur] + price < dp[nxt]:
                        cur_dp[nxt] = min(cur_dp[nxt], dp[cur] + price)
                        q.append(nxt)
            dp = cur_dp
        res = dp[dst]
        if res == float('inf'):
            return -1
        return res
