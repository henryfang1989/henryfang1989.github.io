Solution #1

Use heap to keep the largest K numbers.

Each pop out and push is logk

time complexity: O(nlogk) space complexity: O(k)

Code

import heapq
class Solution(object):
    def findKthLargest(self, nums, k):
        """
        :type nums: List[int]
        :type k: int
        :rtype: int
        """
        h = []
        for i, num in enumerate(nums):
            # keep the largest K numbers in heap
            if i < k:
                heapq.heappush(h, num)
            else:
                heapq.heappushpop(h, num)
        return h[0]

Solution #2

Use quick-select to find the K largest number

The general idea is similar to quicksort. We choose a pivot and partite the numbers based on the pivot.

We use the number of left part of pivot to determine which part should we look into to find the kth largest number. Because left part is smaller than pivot. If left part is more than len(num) - k, we know the kth largest is on left part. Otherwise, it is on right part.

After each partition, we could remove either left part of right part and keep looking into one of part.

Ideally, we could reduce the numbers into half. It turns out time complexity is O(n). The worst case is we remove 1 number each time. It turns out time complexity is O(n^2)

time complexity: O(n), worst O(n^2) space complexity: O(n)

Code

class Solution(object):
    def findKthLargest(self, nums, k):
        """
        :type nums: List[int]
        :type k: int
        :rtype: int
        """
        def partition(nums, left, right, pivot_index):
            store_index = left
            pivot = nums[pivot_index]
            # reserve the position for pivot to allow us to move the pivot
            # back the mid position between left part and right part
            nums[pivot_index], nums[right] = nums[right], nums[pivot_index]
            # move the smaller number to left part
            for i in xrange(left, right):
                if nums[i] < pivot:
                    nums[i], nums[store_index] = nums[store_index], nums[i]
                    store_index += 1
            # move the pivot to the middle position
            nums[right], nums[store_index] = nums[store_index], nums[right]
            return store_index
        
        def select(nums, left, right, k_smallest):
            if left == right:
                return nums[left]
            # always choose mid position value as pivot
            # get the new pivot position after partition
            pivot_index = partition(nums, left, right, (left+right)/2)
            # if the new pivot position is same as k_smallest
            # then the new pivot is the answer
            if pivot_index == k_smallest:
                return nums[k_smallest]
            # if new pivot position is smaller than k_smallest,
            # then the answer on the right side of new pivot position
            elif pivot_index < k_smallest:
                return select(nums, pivot_index+1, right, k_smallest)
            # if new position is larger than k_smallest
            # the answer is on left side of new pivot position
            else:
                return select(nums, left, pivot_index-1, k_smallest)
        
        # instead of find K largest number, we choose len(nums) - K + 1 smallest number
        # because partition() returns the index(position) counting from smallest
        return select(nums, 0, len(nums)-1, len(nums)-k)

Solution #3

Use quick select

The general idea is similar to quicksort. We select a pivot first, then partite the nums into two parts. If right part is more than k numbers, we call the recursion and look for the answer on right part. If right part is less than k numbers, which means the answer is on left part. We call recursion to find the answer on left part. If left and right parts both have less than k numbers, then the mid number is the answer.

time complexity: O(n) Worst: O(n^2) space complexity: O(n)

Code

class Solution(object):
    def findKthLargest(self, nums, k):
        """
        :type nums: List[int]
        :type k: int
        :rtype: int
        """
        def quickselect(nums, start, end, k):
            mid = (start + end) / 2
            pivot = nums[mid]
            left, right = start, end
            while left <= right:
                while left <= right and nums[left] < pivot:
                    left += 1
                while left <= right and nums[right] > pivot:
                    right -= 1
                if left <= right:
                    nums[left], nums[right] = nums[right], nums[left]
                    left += 1
                    right -= 1
            # answer is on right side
            if end - left + 1 >= k:
                return quickselect(nums, left, end, k)
            # answer is on left side
            if end - right < k:
                return quickselect(nums, start, right, k-(end-right))
            # answer is in the middle
            else:
                return nums[(left+right)/2]
        return quickselect(nums, 0, len(nums)-1, k)
