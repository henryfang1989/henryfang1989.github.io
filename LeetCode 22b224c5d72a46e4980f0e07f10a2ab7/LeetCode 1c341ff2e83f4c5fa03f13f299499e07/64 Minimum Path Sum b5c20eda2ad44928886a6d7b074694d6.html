Solution #1

Use DP to solve this issue.

Use dp[i][j] to represent the minPathSum ends with grid[i][j].

In order to get the minimum dp[i][j], we could use the following formula.

dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]

Because there are two paths reaching to the grid[i][j], and we already know minPathSum to grid[i-1][j] is dp[i-1][j] and minPathSum to grid[i][j-1] is dp[i][j-1]. Therefore, the minPathSum for grid[i][j] is the min(dp[i-1][j], dp[i][j-1]) + grid[i][j]

In this solution, we reuse the grid matrix as dp if this question allows us to modify the input.

time complexity: O(m*n) space complexity: O(m*n)

Code

class Solution(object):
    def minPathSum(self, grid):
        """
        :type grid: List[List[int]]
        :rtype: int
        """
        # use DP. In order to get minimum dp[i][j],
        # we have to choose min(dp[i-1][j], dp[i][j-1]) and plus the grid[i][j]
        # we also have to deal with the boundary cases
        for i in xrange(len(grid)):
            for j in xrange(len(grid[0])):
                if i == 0 and j == 0:
                    continue
                elif i == 0:
                    grid[i][j] += grid[i][j-1]
                elif j == 0:
                    grid[i][j] += grid[i-1][j]
                else:
                    grid[i][j] += min(grid[i-1][j], grid[i][j-1])
        return grid[len(grid)-1][len(grid[0])-1]

Solution #2

Use DFS to find the minPathSum

  • Create a helper function to return the minPathSum ending with [i, j]
  • There are two directions can reach to [i, j], the way to calculate the minPathSum ending with [i, j] is grid[i][j] + min(helper(grid, i-1, j), helper(grid, i, j-1))
  • Have to consider the corner cases, which are the boundary nodes.

This solution turns out to TLE because there are so many duplicated calculations

time complexity: O(2^max(m,n) space complexity: O(max(m,n))

Code

class Solution(object):
    def minPathSum(self, grid):
        """
        :type grid: List[List[int]]
        :rtype: int
        """
        # helper function returns the minPathSum ending with [i,j]
        def helper(grid, i, j):
            if i == 0 and j == 0:
                return grid[i][j]
            if j == 0:
                return grid[i][j] + helper(grid, i-1, j)
            if i == 0:
                return grid[i][j] + helper(grid, i, j-1)
            return grid[i][j] + min(helper(grid, i-1, j), helper(grid, i, j-1))
        return helper(grid, len(grid)-1, len(grid[0])-1)

Solution #3

Memorization search

  • This approach is similar to DFS solution
  • We use memorization search to avoid the duplicated search
  • All other logic is same as DFS solution

time complexity: O(m*n) space complexity: O(m*n)

Code

class Solution(object):
    def minPathSum(self, grid):
        """
        :type grid: List[List[int]]
        :rtype: int
        """
        # helper function returns the minPathSum ending with [i,j]
        def helper(grid, i, j, dp):
            if dp[i][j] != -1:
                return dp[i][j]
            if i == 0 and j == 0:
                dp[i][j] = grid[i][j]
            elif j == 0:
                dp[i][j] = grid[i][j] + helper(grid, i-1, j, dp)
            elif i == 0:
                dp[i][j] = grid[i][j] + helper(grid, i, j-1, dp)
            else:
                dp[i][j] = grid[i][j] + min(helper(grid, i-1, j, dp), helper(grid, i, j-1, dp))
            return dp[i][j]
        m, n = len(grid), len(grid[0])
        dp = [[-1 for j in xrange(n)] for i in xrange(m)]
        return helper(grid, m-1, n-1, dp)
