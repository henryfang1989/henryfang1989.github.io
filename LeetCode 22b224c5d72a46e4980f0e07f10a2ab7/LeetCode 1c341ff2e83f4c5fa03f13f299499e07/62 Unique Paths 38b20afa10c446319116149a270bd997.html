Solution #1

DFS to find all possible paths

  1. Treat the uniquePaths to return the number of total paths reaching to [i, j]
  1. There are two way can reach to position [i, j], which [i-1, j] and [i, j-1]
  1. Let’s say we have the answer for uniquePaths(i-1, j) and uniquePaths(i, j-1), then we know uniquePaths(i, j) = uniquePaths(i-1, j) + uniquePaths(i, j-1)
  1. Be careful about the corner cases. The argument passed to uniquePaths are number of points in grid, not the index. Therefore, we should use 1 instead of 0 to check if it reaches to the start point.

time complexity: O(2^(max(m,n))) space complexity: O(max(m, n))

Code

class Solution(object):
    def uniquePaths(self, m, n):
        """
        :type m: int
        :type n: int
        :rtype: int
        """
        if m == 1 and n == 1:
            return 1
        if m == 1:
            return self.uniquePaths(m, n-1)
        if n == 1:
            return self.uniquePaths(m-1, n)
        return self.uniquePaths(m-1, n) + self.uniquePaths(m, n-1)

Solution #2

Use memorization search

This solution is similar to solution 1. But it uses memorization search to avoid the duplicated calculation.

time complexity: O(m*n) space complexity: O(m*n)

Code

class Solution(object):
    def uniquePaths(self, m, n):
        """
        :type m: int
        :type n: int
        :rtype: int
        """
        def helper(m, n, dp):
            if dp[m][n] != -1:
                return dp[m][n]
            if m == 1 and n == 1:
                dp[m][n] = 1
            elif m == 1:
                dp[m][n] = helper(m, n-1, dp)
            elif n == 1:
                dp[m][n] = helper(m-1, n, dp)
            else:
                dp[m][n] = helper(m-1, n, dp) + helper(m, n-1, dp)
            return dp[m][n]
        dp = [[-1 for i in xrange(n+1)] for j in xrange(m+1)]
        return helper(m, n, dp)

Solution #3

Use DP to solve this issue.

  • dp[i][j] represents the uniquePaths ending with [i, j]
  • In order to get dp[i][j], there two ways to get to [i, j], which are [i-1, j] and [i, j-1]
  • Formula: dp[i]][j] = dp[i-1][j] + dp[i][j-1]
  • Create a dp matrix to save values. Be careful about the dp initialization. In order to prevent dealing with the corner cases, which are the nodes in boundary, we expand the dp with one row and one col. We don’t use 0 as starting index.

time complexity: O(m*n) space complexity: O(m*n)

Code

class Solution(object):
    def uniquePaths(self, m, n):
        """
        :type m: int
        :type n: int
        :rtype: int
        """
        dp = [[0 for j in xrange(n+1)] for i in xrange(m+1)]
        # initialize the dp
        dp[0][1] = 1
        for i in xrange(1, m+1):
            for j in xrange(1, n+1):
                dp[i][j] = dp[i-1][j] + dp[i][j-1]

Solution #4

This approach is similar to solution3. But it does some optimization in extra space.

In solution3, we use extra whole matrix to save the dp.

Actually, it is not required. Because it only requires two positions to calculate the current value dp[i][j]

Formula: dp[i][j] = dp[i-1][j] + dp[i][j-1]

We could just create an one-dimension array to save dp[i-1] row and reuse the array.

time complexity: O(m*n) space complexity: O(n)

Code

class Solution(object):
    def uniquePaths(self, m, n):
        """
        :type m: int
        :type n: int
        :rtype: int
        """
        # reuse the dp because dp[i][j] only requires two positions
        dp = [0 for i in xrange(n+1)]
        for i in xrange(1, m+1):
            for j in xrange(1, n+1):
                if i == 1 and j == 1:
                    dp[j] = 1
                else:
                    dp[j] += dp[j-1]
        return dp[-1]
